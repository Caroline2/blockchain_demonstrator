Blockchain

Why is it so interesting?

Blockchain is a modern technology that has existed since the ninetees. The main use came from Satoshi Nakamoto and Bitcoin, however it is also used in many other fields such as electricity commerce. Let's read something about how it actually works.

Ledger.

Imagine blockchain like an open ledger. All the users of the network can add or verify data but no one can modify them.

Distributed network.

Realize the network is actually distributed - no authority, no bank, no server, no central point. All the users (nodes) of the network have the same rights and communicate directly with each other.

Blocks.

One entry of the ledger is not a simple transaction, but a block. Each block contains several transactions and some additional data as a timestamp, in order to be unique.

Chain of blocks.

How can those blocks be chained? In order to chain the blocks, blockchain uses asymmetric cryptography. Each block has it's own hash (a short, unique string representing the block contents) and hash of the previous block. Hash function is not inverse (hash is created from input data but input data can't be created from hash). So if an attacker changes data in the block changes, the hash of such block's data also changes and doesn't match the previous hash of the following block. The network users see that and reject the action.

Genesis block.

The very fist block of the blockchain is special, it's called the Genesis block. The Genesis block does not have a previous block, so it's previous hash is a bunch of zeros. The height of the Genesis block is equal to zero. Then the height of gradually grows with each block (it is unique for each block). Sometimes the Genesis block doesn't even hold any transactions.

Consensus.

If it were easy to add blocks and there was a network error, it would be easy for the attacker to change something in the block and then re-hash the entire blockchain. So when a new block is about to be added, the user who wants to add (mine) it must prove he/she has put enough effort in it (eg his/her computing power). That ensures that the potential fraud does not pay off for the attacker. It also encourages the users to add new blocks â€“ the node (miner) who meets the network requirements for addition of the block first receives a reward in the form of network tokens.

Forks.

The order of the blocks in the blockchain is based on their timestamps, ie the time each block was added to the blockchain. Sometimes it happens that two blocks are added at the same time. In such a case it needs to be judged by which branch the blockchain will continue. One of the basic algorithms is the longest chain rule. The network waits for the next block which is then chained to one of the two controversial blocks. The branch the very new block was added to is the longest, so the the direction is clearly defined and the other controversial block can be discarded.

Transaction authorization.

Blockchain transactions need authorization. Otherwise, each node could credit the tokens of another node without it's permission. To guarantee this, asymmetric cryptography is used again. Each node has it's own private and public key. Using a private key, it creates a unique digital signature for each transaction that clearly identifies it (but does not disclose the private key of).

Double spending.

Double spending is a problem where one user wants to send the same tokens to two recipients. In practice, it is usually handled by a central point (a bank that records which tokens who sent to whom and when). However, there is no central authority in the blockchain, therefore. However, all transfers are recorded (an open ledger, remember?). So if user X tries to send token # 120 to user Y and a second later to user Z, the network will already know that token # 120 belongs to user Y and will reject the transaction.

51 % Attack.

51 % Attack is a problem where one user owns 51 % of the network (ie 51 % of the computational power or 51 % of the network tokens etc) and so is able to verify his fraudulent transactions. If he changed something in the previous blocks and had to re-hash the blockchain, we know it wouldn't pay off. So the only real option seems to be to attach a fraudulent transaction to a block that is currently being mined. That would pass. Therefore, it is very important that network resources are not distributed very unevenly.

Anonymity.

Blockchain is often mistaken by the public for being completely anonymous. This is not true from the essence of transaction registration - if a given blockchain does not use any software that would try to hide the addresses of individual users (address mixing), it is very easy to find out how many tokens have ever come to which account. This fact has helped to detect many fraudsters in dark markets. In general, it can be said that blockchain is one of the more anonymous technologies, but it is definitely not completely anonymous.

Private blockchains.

With the growing popularity of the technology, many companies or schools are implementing their own private blockchains. However, in this case, the very essence of the blockchain, ie the distributed network, disappears, because the company or school is its owner and administrator, and has greater privileges, which contradicts the essence of blockchain. So, the use of blockchain technology in this way can bring some technical benefits to their owners, but they are definitely different from the main purpose for which the blockchain is used.

Blockchain & electricity

How can blockchain help the electricity commerce?

Blockchain is used mainly with cryptocurrencies, but also in other sectors such as electricity trade. So what are the advantages blockchain can bring in here?

Blockchain in electricity commerce is most often used in the network of local retailers, who provide each other with electricity from their solar panels. Both entry and the exit from the sales network is free, which motivates the connection of more and more sellers. They can set their own prices.
The distance between the buyer and the seller is also crucial for the sale. While electricity normally travels to households over long wires and through other devices, in a distributed system we can easily sell electricity to our neighbors, which will greatly reduce transportation costs and thus significantly reduce the overall price of electricity, which attracts new customers. The buyer thus negotiates directly with the seller (no third party or authoritative seller), has a lot more offers to choose from, which he can compare, and with his purchase supports, for example, someone he knows and trusts.

Thanks to the blockchain, the overall approach to electricity sales can change. It also offers the possibility of selling to small suppliers, who are significantly disadvantaged in the current mostly centralized model. However, such a system is still in its infancy. Although Blockchain is a very powerful software tool, the question remains how exactly to solve the transport of electricity itself through transmission networks, which are often problematic, mostly belonging to private individuals.
